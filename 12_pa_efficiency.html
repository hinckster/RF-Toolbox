Power Amplifier / P1dB / Efficiency

PA classes A, B, AB, C: drain efficiency vs output back-off, gain compression, OIP3. PAPR impact on average efficiency.

Efficiency vs Back-off
η_A = Pout / (2·Psat) · η_B = (π/4)·√(Pout/Psat)
Back-off (BO) = Psat − Pout [dB]. Class A peaks at 50%, Class B at 78.5%.
Gain Compression (Saleh model)
Pout ≈ G·Pin − (G·Pin)³/(3·P1dB²) [soft compression]
P1dB(in) = P1dB(out) − G + 1 dB
At P1dB: actual gain is 1 dB below small-signal gain G
OIP3 & PAPR
OIP3 ≈ P1dB(out) + 9.6 dB [rule of thumb]
η_avg = η_peak · Pout_avg/Pout_peak
OFDM: PAPR 8–12 dB. Must back off from P1dB to avoid compression.

► Things to Try
  • Switch between Class A and Class B with the same Pop and P1dB: observe Class B delivers ~57% efficiency vs Class A's ~25% at 6 dB back-off — the tradeoff that drives Doherty amplifier designs.
  • Load the "WiFi PA" preset (PAPR = 8 dB, Class A): average η drops to ~6%. Switch to Class B — average η improves to ~22%, showing why modern 802.11ax uses envelope-tracking PAs.
  • Increase Pop until back-off drops below 1 dB and observe the gain compression warning — this is the saturated PA regime used in constant-envelope modulations like GMSK.

Power Amplifier / P1dB / Efficiency — Help
What this tool calculates
Models PA drain efficiency η vs output back-off for Classes A, AB, B, and C. Class A: η = Pout/(2·Psat), peaking at 50%. Class B: η = (π/4)·√(Pout/Psat), peaking at 78.5%. Gain compression is modeled via a Saleh AM-AM softlimiter referenced to P1dB. OIP3 is estimated as P1dB(out) + 9.6 dB. Average efficiency under a modulated waveform is computed using PAPR: the average power is Pop − PAPR(dB), and ηavg = η(Pavg).
Sliders & inputs
  • Small-signal gain G (5–40 dB): Linear (uncompressed) voltage gain of the PA. Determines the input power required to reach a given output level.
  • Output P1dB (10–55 dBm): Output power at the 1 dB gain compression point. Psat is modeled as P1dB + 2 dB. Typical values: handset PA ~30 dBm, basestation ~47 dBm.
  • Operating Pout (0–55 dBm): The actual output power at the intended operating point. Back-off = P1dB − Pop; a larger back-off improves linearity but reduces efficiency.
  • Vdd supply (1–50 V): DC supply voltage — shown for reference; used to contextualize DC power draw.
  • PAPR (0–15 dB): Peak-to-average power ratio of the modulated signal. CW = 0 dB; OFDM typically 8–12 dB; 64QAM OFDM commonly requires ~10 dB.
  • PA Class tabs (A / AB / B / C): Switches the conduction-angle model. Class A conducts 360°; B conducts 180°; C <180° (high efficiency, nonlinear); AB is a practical compromise.
How to read the chart
The left chart (Efficiency vs Output Power) plots drain efficiency (%) against Pout in dBm over a 20 dB dynamic range below Psat. The selected class is the solid blue line; Class A and B references are dashed for comparison. The orange dot marks the operating point. The right chart (Gain Compression / AM-AM) plots actual Pout vs Pin; the dashed grey line is the ideal linear response; the red dashed line is P1dB; the orange dot is the operating point. Deviation between actual and linear curves indicates how much gain compression is occurring.
Key results
  • Back-off from P1dB: dB margin between operating power and the compression point. >6 dB is generally considered linear operation.
  • Peak efficiency η: Maximum theoretical efficiency for the selected class (50% for A, 78.5% for B).
  • η at operating Pout: Drain efficiency at the chosen Pop.
  • Avg η (with PAPR): Long-term average efficiency accounting for signal envelope statistics.
  • DC power draw: Estimated PDC = Pop / ηop.
  • OIP3 / IIP3: Third-order intercept points estimated from the P1dB + 9.6 dB rule of thumb.
  • Gain at operating point: Compressed gain Gop from the Saleh model — always ≤ small-signal gain G.
