Superheterodyne Receiver

Frequency plan: RF → IF conversion. Image rejection vs IF frequency. Double-conversion. Spurious LO harmonics.

Frequency Plan
f_LO = f_RF ± f_IF
f_image = f_RF ± 2·f_IF (upper LO)
f_image = f_RF − 2·f_IF (lower LO)
Two mixing sidebands — one is the desired IF, one is image. Image must be filtered before mixer.
Image Rejection
IRR = 20 log₁₀(2·f_IF / BW_RF)
BW_RF = RF preselect filter bandwidth
Higher IF → better image separation but harder channel filter. Key design tradeoff (Razavi 2.4).
Spurious Responses
f_spur = (m·f_LO ± f_IF) / n
Common: 2·f_LO − f_IF, 3·f_LO − 2·f_IF
LO harmonic mixing. These land in-band and cannot be filtered. Pozar Ch. 9.

► Things to Try
  • Load the AM radio preset (1 MHz RF, 455 kHz IF) — notice the image lands at 1.910 MHz, only 910 kHz away from the desired signal. Now increase f_IF to 10.7 MHz (FM standard) and watch image separation and IRR improve dramatically.
  • Switch to Double conversion and set f_IF2 to 455 kHz while leaving f_IF at 10.7 MHz — this replicates the classic FM receiver architecture: high first IF for image rejection, low second IF for a sharp channel filter.
  • With the GSM 900 preset, tighten RF filter BW from 20 MHz to 1 MHz and observe how IRR jumps by ~26 dB — demonstrating why tight preselection is critical at microwave frequencies where IF choices are constrained.

Superheterodyne Receiver — Help
What this tool calculates
Models the frequency plan of a single- or double-conversion superheterodyne receiver. It computes f_LO = f_RF ± f_IF, locates the image frequency f_image = f_RF ± 2·f_IF, and calculates the ideal image rejection ratio IRR = 20·log₁₀(2·f_IF / BW_RF) in dB. In double-conversion mode a second LO and second IF stage are added, giving a second mixing product and additional image at the second conversion.
Sliders & inputs
  • RF freq f_RF — Desired receive frequency, 10 MHz–6 GHz. Click the unit label to toggle between MHz and GHz. This is the signal you want to down-convert.
  • 1st IF f_IF — First intermediate frequency, 1–1000 MHz. Higher IF pushes the image further from the RF passband, improving image rejection. Lower IF makes the channel filter easier to build.
  • RF filter BW — Bandwidth of the RF preselector (band-pass filter before the mixer), 0.1–200 MHz. Narrower bandwidth rejects the image signal more strongly; this is the denominator of the IRR formula.
  • Conversion mode — Toggle between Single and Double conversion. In double-conversion mode a second IF slider appears (f_IF2, 0.1–100 MHz), enabling a second down-conversion stage for additional selectivity.
  • 2nd IF f_IF2 (double-conversion only) — Second intermediate frequency, typically 10.7 MHz or 455 kHz. This stage provides the narrow channel filter while the first IF provides image rejection.
How to read the charts
Top chart — Frequency Spectrum: A bar chart showing the key frequency components as vertical bars. Blue = RF signal; yellow = LO frequency; purple = IF output; red = image frequency. Bar height represents relative amplitude (normalized). The separation between the RF bar and the image bar is 2·f_IF — narrower separation means harder image rejection.

Bottom chart — Image Rejection vs IF Frequency: X-axis is IF frequency (MHz, log scale); Y-axis is IRR (dB). The curve shows how IRR improves as IF increases relative to the RF filter bandwidth. A vertical marker indicates the currently selected IF. The horizontal dashed line at 40 dB marks a common minimum design target.
Key results
  • LO freq — Local oscillator frequency (f_RF + f_IF for high-side injection, f_RF − f_IF for low-side). The tool uses high-side injection by default.
  • Image freq — The unwanted input frequency that also mixes to f_IF. Any signal at this frequency passes through the mixer and corrupts the desired channel.
  • Image sep. — Frequency distance between the RF signal and the image: 2·f_IF. Must be wider than the RF preselect filter to achieve useful rejection.
  • IRR (ideal) — Ideal image rejection ratio in dB assuming a single-pole RF filter with the specified bandwidth. Real filters achieve this only near band-center.
